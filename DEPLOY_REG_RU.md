# Подробная инструкция по деплою на хостинг Reg.ru (без виртуальной машины)

Пошаговое руководство по публикации проекта «Система трудовых отрядов» на **Shared-хостинге Reg.ru** (общий хостинг, не VPS и не выделенный сервер).

---

## Что понадобится

- Аккаунт на [Reg.ru](https://www.reg.ru/) с оплаченным **хостингом** (тариф «Хостинг» или аналог).
- Домен, привязанный к этому хостингу (или поддомен).
- На компьютере: **Node.js** 18+ и **npm** (или pnpm).

---

## Часть 1. Подготовка проекта на компьютере

### 1.1. Установка зависимостей и сборка

В корне проекта (там, где лежит `package.json`) выполните в терминале:

```bash
npm install
npm run build
```

Должна появиться папка **`dist`**. В ней:

- `index.html` — главная страница;
- `assets/` — собранные JS и CSS;
- `api/` — копия папки `public/api/` (все PHP-скрипты и подпапки).

### 1.2. Проверка содержимого `dist`

Убедитесь, что в `dist` есть:

```
dist/
├── index.html
├── assets/
│   ├── index-*.js
│   └── index-*.css
└── api/
    ├── upload.php
    ├── upload_to_yandex.php
    ├── get_yandex_file.php
    ├── save_data.php
    ├── uploads/          (может быть пустой, внутри желательно .htaccess)
    └── data/             (можно не создавать — создастся при первой заявке)
```

Если папок `api/uploads` или `api/data` нет — их создаст PHP при первом запросе (или создайте пустые вручную).

## Часть 2. Вход в панель Reg.ru и определение корня сайта

### 2.1. Вход в панель управления

1. Зайдите на [https://www.reg.ru/](https://www.reg.ru/) и авторизуйтесь.
2. Откройте раздел **«Хостинг»** → выберите нужный хостинг-аккаунт.
3. Перейдите в **«Файловый менеджер»** (или «Управление хостингом» → «Файлы»).

### 2.2. Корневая папка сайта

На Reg.ru корень сайта обычно одна из папок:

- **`public_html`** — для основного домена;
- **`www`** — иногда домен привязан сюда;
- **`subdomain.domain.ru`** — для поддомена (папка с именем поддомена).

Запомните, в какую папку нужно заливать файлы. Далее в инструкции эта папка называется **«корень сайта»**.

---

## Часть 3. Загрузка файлов на хостинг

### Вариант А: Через файловый менеджер Reg.ru (ZIP)

1. На своём компьютере откройте папку **`dist`**.
2. Выделите **все файлы и папки внутри** `dist` (не саму папку `dist`):
   - `index.html`
   - папка `assets`
   - папка `api`
3. Создайте **ZIP-архив** из выделенного (например, `site.zip`).
4. В файловом менеджере Reg.ru откройте **корень сайта** (`public_html` или аналог).
5. Нажмите **«Загрузить»** / **«Upload»** и выберите `site.zip`.
6. После загрузки найдите архив в списке файлов и выберите **«Распаковать»** / **«Extract»**.
7. Убедитесь, что в корне появились `index.html`, папки `assets` и `api`. При необходимости переместите содержимое из вложенной папки в корень (если распаковалось в подпапку).
8. Удалите пустой архив `site.zip`, чтобы не засорять корень.

### Вариант Б: Через FTP/SFTP

1. В панели Reg.ru найдите данные для **FTP** или **SFTP**: хост (например, `ftp.ваш-домен.ru` или `ваш-логин.regru.ru`), логин, пароль. Порт FTP — 21, SFTP — 22.
2. Подключитесь с компьютера программой FileZilla, WinSCP или аналогом.
3. На сервере откройте **корень сайта** (`public_html` и т.п.).
4. С локального диска откройте папку **`dist`**.
5. Перенесите в корень сайта **все содержимое** `dist`:
   - перетащите `index.html`, папки `assets` и `api` в корень;
   - не создавайте лишнюю папку `dist` на сервере.

Итог на хостинге должен быть таким:

```
корень сайта (public_html и т.п.)/
├── index.html
├── assets/
│   ├── index-xxxxx.js
│   └── index-xxxxx.css
└── api/
    ├── upload.php
    ├── upload_to_yandex.php
    ├── get_yandex_file.php
    ├── save_data.php
    ├── uploads/
    └── data/
```

## Часть 4. Права на папки и файлы

Чтобы PHP мог создавать файлы и папки:

1. В файловом менеджере откройте папку **`api`**.
2. Для папок **`uploads`** и **data** установите права **755** (если папки создаются скриптами при первом запросе — выставите 755 после их появления).
3. Для PHP-файлов (`.php`) обычно достаточно **644**.

В интерфейсе Reg.ru: правый клик по папке/файлу → **«Права доступа»** / **«Chmod»** → укажите 755 для папок, 644 для файлов.

## Часть 5. Настройка Яндекс.Диска (по желанию)

Если нужна загрузка документов на Яндекс.Диск:

1. Получите OAuth-токен приложения с правом `cloud_api:disk.write` ([Яндекс OAuth](https://oauth.yandex.com/)).
2. На хостинге в папке **`api`** создайте файл **`.yandex_oauth_token`** (имя точно с точкой в начале).
3. Внутри файла — одна строка: ваш OAuth-токен, без кавычек и пробелов.
4. Установите права на файл **600** (доступ только владельцу). Файл не должен быть доступен по прямой ссылке (положите его только в `api/`, не в веб-корень).

Альтернатива: в панели Reg.ru, если есть раздел «Переменные окружения», задайте переменную **`YANDEX_OAUTH_TOKEN`** со значением токена. Тогда файл `.yandex_oauth_token` не обязателен.

Без токена заявки будут сохраняться, но файлы не попадут на Диск (форма может отправиться без вложений или с сохранением только в localStorage на стороне клиента, в зависимости от настроек).

## Часть 6. Проверка работы сайта

1. Откройте в браузере ваш домен (например, `https://ваш-домен.ru`).
2. Должна открыться главная страница приложения «Трудовое лето».
3. Пролистайте до формы регистрации, заполните поля, выберите смену, при необходимости прикрепите файл и отправьте заявку.
4. Проверьте:
   - появилось ли сообщение об успешной отправке;
   - в личном кабинете (после входа по email и паролю) отображается ли заявка;
   - в админ-панели (кнопка «Админ. панель», пароль задаётся в коде) отображаются ли заявки и файлы.

Если заявки не сохраняются:

- Убедитесь, что в корне сайта действительно есть папка `api` с файлами `save_data.php`, `upload.php` и т.д.
- Проверьте права на `api/data` (755) и при необходимости создайте папку `api/data` вручную.
- Откройте в браузере инструменты разработчика (F12) → вкладка «Сеть» и посмотрите, не возвращают ли запросы к `/api/save_data.php` ошибку (4xx/5xx).

---

## Часть 7. HTTPS и домен

- На Reg.ru обычно можно включить **бесплатный SSL** (Let's Encrypt) в разделе «SSL» / «Безопасность» панели хостинга. После этого сайт будет открываться по `https://`.
- Убедитесь, что домен привязан к нужной папке (например, `public_html`). Это настраивается в разделе «Домены» / «Сайты» панели Reg.ru.

---

## Часть 8. Обновление сайта после изменений в коде

1. На компьютере в корне проекта снова выполните:
   ```bash
   npm run build
   ```
2. Заново залейте в **корень сайта** содержимое папки `dist` (перезаписав `index.html`, папки `assets` и `api`).
3. Файл **`.yandex_oauth_token`** и папки `api/uploads`, `api/data` при перезаписи не трогайте (или не включайте их в архив при обновлении), чтобы не потерять токен и загруженные данные.

Рекомендуется перед перезаписью сделать резервную копию папки `api` на хостинге (скачать архивом), если там уже есть важные данные.

- [ ] Для `api/uploads` и `api/data` выставлены права 755.
- [ ] При использовании Яндекс.Диска: в `api/` создан файл `.yandex_oauth_token` с токеном, права 600.

При соблюдении этих шагов проект корректно работает на Shared-хостинге Reg.ru без виртуальной машины.
