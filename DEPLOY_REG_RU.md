# Деплой на хостинг Reg.ru (Shared, без VPS)

Пошаговое руководство по публикации проекта «Система трудовых отрядов» на **Shared-хостинге** (например, Reg.ru): общий хостинг, не VPS и не выделенный сервер.

---

## Что понадобится

- Аккаунт на [Reg.ru](https://www.reg.ru/) с оплаченным **хостингом**.
- Домен или поддомен, привязанный к этому хостингу.
- На компьютере: **Node.js** 18+ и **npm** (или pnpm).

---

## Часть 1. Подготовка проекта на компьютере

### 1.1. Установка зависимостей и сборка

В корне проекта (где лежит `package.json`):

```bash
npm install
npm run build
```

Должна появиться папка **`dist`**. В ней:

- `index.html` — точка входа SPA;
- `assets/` — собранные JS и CSS (основной бандл + отдельный чанк для библиотеки Excel `xlsx`);
- `api/` — копия `public/api/` (все PHP-скрипты и служебные файлы);
- **`.htaccess`** — из `public/`, нужен в **корне сайта** (SPA + проброс заголовка `Authorization` для PHP).

### 1.2. Проверка содержимого `dist`

Убедитесь, что есть:

```
dist/
├── index.html
├── .htaccess
├── assets/
│   ├── index-*.js
│   ├── index-*.js   (дополнительный чанк для xlsx — имя меняется при сборке)
│   └── index-*.css
└── api/
    ├── db_config.php
    ├── db_setup.php
    ├── register.php
    ├── login.php
    ├── get_applications.php
    ├── update_application.php
    ├── get_stats.php
    ├── save_data.php
    ├── upload.php
    ├── upload_to_yandex.php
    ├── get_yandex_file.php
    ├── yandex_oauth_config.php
    ├── yandex_oauth_callback.php
    ├── .user.ini
    ├── uploads/
    └── data/
```

Имя файла `index-*.js` меняется при каждой сборке — ориентируйтесь на актуальные имена в `dist/assets/` и на ссылку в `dist/index.html`.

## Часть 2. Вход в панель Reg.ru и корень сайта

1. Зайдите на [https://www.reg.ru/](https://www.reg.ru/) и авторизуйтесь.
2. Откройте **«Хостинг»** → нужный аккаунт → **«Файловый менеджер»** (или аналог).

Корень сайта обычно:

- `public_html` — основной домен;
- `www` — иногда домен указывает сюда;
- отдельная папка — для поддомена.

Далее эта папка называется **«корень сайта»**.

---

## Часть 3. Загрузка файлов на хостинг

### Вариант А: ZIP через файловый менеджер

1. Откройте локальную папку **`dist`**.
2. Выделите **всё содержимое** `dist` (не саму папку `dist`): `index.html`, `.htaccess`, `assets`, `api`.
3. Упакуйте в **ZIP** (например, `site.zip`).
4. В корне сайта на хостинге загрузите архив и **распакуйте**.
5. Убедитесь, что `index.html`, `.htaccess`, `assets/` и `api/` лежат **в корне сайта**, а не во вложенной папке без причины.

### Вариант Б: FTP/SFTP

Скопируйте **всё содержимое** `dist` в корень сайта. Папку `dist` на сервере создавать не нужно.

---

## Часть 4. База данных MySQL

### 4.1. Создание БД

В панели хостинга: **«Базы данных»** / **MySQL** → создать базу. Запишите:

- имя базы;
- пользователя;
- пароль;
- хост (часто `localhost`).

### 4.2. Файл `api/.db_credentials`

В папке **`api`** на сервере создайте файл **`.db_credentials`** (с точкой). **Ровно 4 строки:**

```
localhost
имя_базы
имя_пользователя
пароль
```

Права на файл: **600**.

### 4.3. Создание и обновление таблиц

Откройте в браузере (один раз):

```
https://ВАШ_ДОМЕН/api/db_setup.php
```

При успехе должно отображаться сообщение о создании/обновлении таблиц (в т.ч. колонка `status` с вариантом `reserve` для резерва заявок).

**Сразу после успешного выполнения удалите `db_setup.php` с сервера**, чтобы не запускать скрипт повторно извне.

### 4.4. Пароль администратора

По умолчанию в коде может использоваться пароль админ-панели из документации; **обязательно замените** его:

- файл **`api/.admin_password`** — одна строка с паролем, права **600**;  
  или **переменная окружения** `ADMIN_PASSWORD` в панели хостинга.

---

## Часть 5. Права, лимиты PHP

- Папки **`api/uploads`** и при необходимости **`api/data`**: права **755** (запись для веб-сервера).
- PHP-файлы: обычно **644**.
- Секретные файлы: **600** (`.db_credentials`, `.admin_password`, `.yandex_oauth_token`, `.yandex_app_credentials`).

В **`api/.user.ini`** заданы увеличенные лимиты для загрузки файлов (`upload_max_filesize`, `post_max_size`, `max_file_uploads` и т.д.). На части хостингов `.user.ini` подхватывается автоматически; если загрузки не проходят, проверьте лимиты в панели PHP или обратитесь в поддержку хостинга.

---

## Часть 6. Настройка Яндекс.Диска (по желанию)

Если нужна загрузка документов на Яндекс.Диск:

### Через страницу `/yandexouth`

1. В [консоли разработчика Яндекса](https://oauth.yandex.com/) создайте приложение (веб-сервис).
2. **Redirect URI:** `https://ВАШ_ДОМЕН/api/yandex_oauth_callback.php`.
3. В **`api`** создайте **`.yandex_app_credentials`**: строка 1 — Client ID, строка 2 — Client secret. Права **600**.
4. Откройте **`https://ВАШ_ДОМЕН/yandexouth`**, пройдите авторизацию — токен сохранится (файл `.yandex_oauth_token` в `api/`).

### Вручную

Создайте **`api/.yandex_oauth_token`** — одна строка с OAuth-токеном, права **600**.

Без токена заявки в БД могут сохраняться, но файлы на Диск не попадут (или используется локальный `upload.php` в `api/uploads/`).

## Часть 7. Проверка работы

1. Откройте домен — главная страница.
2. Проверьте **регистрацию** и **вход** в личный кабинет.
3. Проверьте **админ-панель** (пароль из `.admin_password` или `ADMIN_PASSWORD`).
4. **Блок «Свободные места»** использует **`/api/get_stats.php`** — при ошибке БД цифры могут быть нулевыми; смотрите ответ в Network (F12).
5. Маршрут **`/yandexouth`** должен открываться (SPA: для этого в корне должен лежать корневой **`.htaccess`** из сборки).

Если после входа в личный кабинет данные не подгружаются, проверьте, что Apache **не обрезает** заголовок `Authorization` — для этого используется **`/.htaccess`** из `dist`.

---

## Часть 8. HTTPS и домен

Включите бесплатный SSL (Let's Encrypt) в панели хостинга, если доступно. Убедитесь, что домен указывает на нужную папку.

---

## Часть 9. Обновление сайта после правок в коде

1. Локально:
   ```bash
   npm run build
   ```
2. Загрузите в **корень сайта** новые **`index.html`**, папку **`assets/`** целиком, папку **`api/`** (PHP-файлы).  
   Файл **`.htaccess`** в корне обновите, если он менялся в репозитории.

**Не перезаписывайте на сервере без необходимости:**

- `api/.db_credentials`
- `api/.yandex_oauth_token`
- `api/.yandex_app_credentials`
- `api/.admin_password`
- содержимое **`api/uploads/`** (загруженные файлы пользователей)
