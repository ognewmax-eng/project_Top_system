# Подробная инструкция по деплою на хостинг Reg.ru (без виртуальной машины)

Пошаговое руководство по публикации проекта «Система трудовых отрядов» на **Shared-хостинге Reg.ru** (общий хостинг, не VPS и не выделенный сервер).

---

## Что понадобится

- Аккаунт на [Reg.ru](https://www.reg.ru/) с оплаченным **хостингом** (тариф «Хостинг» или аналог).
- Домен, привязанный к этому хостингу (или поддомен).
- На компьютере: **Node.js** 18+ и **npm** (или pnpm).

---

## Часть 1. Подготовка проекта на компьютере

### 1.1. Установка зависимостей и сборка

В корне проекта (там, где лежит `package.json`) выполните в терминале:

```bash
npm install
npm run build
```

Должна появиться папка **`dist`**. В ней:

- `index.html` — главная страница;
- `assets/` — собранные JS и CSS;
- `api/` — копия папки `public/api/` (все PHP-скрипты и подпапки).

### 1.2. Проверка содержимого `dist`

Убедитесь, что в `dist` есть:

```
dist/
├── index.html
├── assets/
│   ├── index-*.js
│   └── index-*.css
└── api/
    ├── db_config.php
    ├── db_setup.php
    ├── register.php
    ├── login.php
    ├── get_applications.php
    ├── update_application.php
    ├── save_data.php
    ├── upload.php
    ├── upload_to_yandex.php
    ├── get_yandex_file.php
    ├── uploads/
    └── data/
```

## Часть 2. Вход в панель Reg.ru и определение корня сайта

### 2.1. Вход в панель управления

1. Зайдите на [https://www.reg.ru/](https://www.reg.ru/) и авторизуйтесь.
2. Откройте раздел **«Хостинг»** → выберите нужный хостинг-аккаунт.
3. Перейдите в **«Файловый менеджер»** (или «Управление хостингом» → «Файлы»).

### 2.2. Корневая папка сайта

На Reg.ru корень сайта обычно одна из папок:

- **`public_html`** — для основного домена;
- **`www`** — иногда домен привязан сюда;
- **`subdomain.domain.ru`** — для поддомена (папка с именем поддомена).

Запомните, в какую папку нужно заливать файлы. Далее в инструкции эта папка называется **«корень сайта»**.

---

## Часть 3. Загрузка файлов на хостинг

### Вариант А: Через файловый менеджер Reg.ru (ZIP)

1. На своём компьютере откройте папку **`dist`**.
2. Выделите **все файлы и папки внутри** `dist` (не саму папку `dist`):
   - `index.html`
   - папка `assets`
   - папка `api`
3. Создайте **ZIP-архив** из выделенного (например, `site.zip`).
4. В файловом менеджере Reg.ru откройте **корень сайта** (`public_html` или аналог).
5. Нажмите **«Загрузить»** / **«Upload»** и выберите `site.zip`.
6. После загрузки найдите архив в списке файлов и выберите **«Распаковать»** / **«Extract»**.
7. Убедитесь, что в корне появились `index.html`, папки `assets` и `api`. При необходимости переместите содержимое из вложенной папки в корень (если распаковалось в подпапку).
8. Удалите пустой архив `site.zip`, чтобы не засорять корень.

### Вариант Б: Через FTP/SFTP

1. В панели Reg.ru найдите данные для **FTP** или **SFTP**: хост (например, `ftp.ваш-домен.ru` или `ваш-логин.regru.ru`), логин, пароль. Порт FTP — 21, SFTP — 22.
2. Подключитесь с компьютера программой FileZilla, WinSCP или аналогом.
3. На сервере откройте **корень сайта** (`public_html` и т.п.).
4. С локального диска откройте папку **`dist`**.
5. Перенесите в корень сайта **все содержимое** `dist`:
   - перетащите `index.html`, папки `assets` и `api` в корень;
   - не создавайте лишнюю папку `dist` на сервере.

## Часть 4. Настройка базы данных MySQL

### 4.1. Создание базы данных на Reg.ru

1. В панели управления хостингом перейдите в раздел **«Базы данных»** (или «MySQL»).
2. Нажмите **«Создать базу данных»**.
3. Запишите:
   - **Имя базы** (например, `u1234567_top`)
   - **Имя пользователя** (например, `u1234567_topuser`)
   - **Пароль** — установите надёжный пароль
   - **Хост** — обычно `localhost` (на Reg.ru так и есть)

### 4.2. Создание файла с настройками подключения

На хостинге в папке **`api`** создайте файл **`.db_credentials`** (с точкой в начале). Внутри — **4 строки**:

```
localhost
u1234567_top
u1234567_topuser
ВашПарольОтБазы
```

Строки по порядку:
1. Хост (обычно `localhost`)
2. Имя базы данных
3. Имя пользователя
4. Пароль

Установите права на файл: **600** (чтобы только владелец мог читать).

### 4.3. Создание таблиц

Откройте в браузере:

```
https://ВАШ_ДОМЕН/api/db_setup.php
```

Если всё настроено правильно, появится зелёная надпись: **«Таблицы успешно созданы!»**

Будут созданы две таблицы:
- **`users`** — логины и пароли (пароли хранятся зашифрованными через bcrypt)
- **`applications`** — заявки участников

**После успешного создания таблиц удалите файл `db_setup.php` с хостинга**, чтобы никто не мог запустить его повторно.

### 4.4. Пароль администратора (по желанию)

По умолчанию пароль админ-панели: `admin123`. Чтобы сменить его:

Создайте в папке **`api`** файл **`.admin_password`** — внутри одна строка с новым паролем. Права: **600**.

Или в панели Reg.ru задайте переменную окружения **`ADMIN_PASSWORD`**.

## Часть 5. Права на папки и файлы

Чтобы PHP мог создавать файлы и папки:

1. В файловом менеджере откройте папку **`api`**.
2. Для папок **`uploads`** и **`data`** установите права **755**.
3. Для PHP-файлов (`.php`) обычно достаточно **644**.
4. Для секретных файлов (`.db_credentials`, `.yandex_oauth_token`, `.yandex_app_credentials`, `.admin_password`) — **600**.

В интерфейсе Reg.ru: правый клик по папке/файлу → **«Права доступа»** / **«Chmod»** → укажите нужное значение.

## Часть 6. Настройка Яндекс.Диска (по желанию)

Если нужна загрузка документов на Яндекс.Диск, можно получить токен вручную или через страницу на сайте.

### Вариант А: получение токена через страницу /yandexouth

1. В [консоли разработчика Яндекса](https://oauth.yandex.com/) создайте приложение типа «Веб-сервисы».
2. В настройках укажите **Redirect URI**: `https://ВАШ_ДОМЕН/api/yandex_oauth_callback.php`.
3. На хостинге в папке **`api`** создайте файл **`.yandex_app_credentials`** — две строки: первая — **Client ID**, вторая — **Client secret**. Права: **600**.
4. Откройте в браузере **`https://ВАШ_ДОМЕН/yandexouth`**. Нажмите «Войти через Яндекс», разрешите доступ — токен сохранится автоматически.

### Вариант Б: ручное создание токена

1. Получите OAuth-токен приложения с правом доступа к Яндекс.Диску.
2. На хостинге в папке **`api`** создайте файл **`.yandex_oauth_token`** — одна строка: ваш OAuth-токен.
3. Права: **600**.

Без токена заявки будут сохраняться, но файлы не попадут на Диск.

## Часть 7. Проверка работы сайта

1. Откройте в браузере ваш домен (например, `https://ваш-домен.ru`).
2. Должна открыться главная страница приложения «Трудовое лето».
3. Заполните форму регистрации, отправьте заявку.
4. Проверьте:
   - появилось ли сообщение об успешной отправке;
   - в личном кабинете (после входа по email и паролю) отображается ли заявка;
   - в админ-панели отображаются ли заявки и можно ли менять статусы.

Если заявки не сохраняются:

- Убедитесь, что файл `.db_credentials` в папке `api` содержит правильные данные.
- Откройте `https://ВАШ_ДОМЕН/api/db_setup.php` и проверьте, нет ли ошибок подключения к БД.
- В инструментах разработчика браузера (F12 → «Сеть») проверьте ответы от `/api/register.php` и `/api/login.php`.

## Часть 8. HTTPS и домен

- На Reg.ru обычно можно включить **бесплатный SSL** (Let's Encrypt) в разделе «SSL» / «Безопасность» панели хостинга.
- Убедитесь, что домен привязан к нужной папке.

---

## Часть 9. Обновление сайта после изменений в коде

1. На компьютере в корне проекта снова выполните:
   ```bash
   npm run build
   ```
2. Заново залейте в **корень сайта** содержимое папки `dist` (перезаписав `index.html`, папки `assets` и `api`).
3. **НЕ перезаписывайте на хостинге:**
   - `.db_credentials` — настройки БД
   - `.yandex_oauth_token` — токен Яндекса
   - `.yandex_app_credentials` — учётные данные приложения Яндекса
   - `.admin_password` — пароль администратора
   - папку `api/uploads` — загруженные файлы
