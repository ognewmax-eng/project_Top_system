# Система трудовых отрядов — «Трудовое лето»

Веб-приложение для приёма заявок участников программы: регистрация с аккаунтом, загрузка документов (в т.ч. на Яндекс.Диск), личный кабинет со статусом заявки, панель администратора с фильтрами, редактированием заявок и выгрузкой в Excel.

**Пошаговый деплой на Shared-хостинг (Reg.ru):** [DEPLOY_REG_RU.md](DEPLOY_REG_RU.md).

---

## Возможности

| Область | Описание |
|--------|----------|
| **Главная страница** | Герой, блок «Как это работает», свободные места по сменам (данные из БД), форма регистрации |
| **Адаптивная вёрстка** | Мобильные экраны: хук `useIsMobile`, бургер-меню в шапке, сетки и отступы под узкие экраны |
| **Регистрация** | Email/пароль, личные данные, льготы, смена, обязательные документы (см. ниже), согласие на обработку ПД |
| **Шаблоны документов** | Ссылка на [Яндекс.Диск с шаблонами](https://disk.yandex.ru/d/tq_-HiC6wB7wRg) в блоке «Документы» |
| **Личный кабинет** | Статусы заявки (в т.ч. «в резерве»), данные заявки, доработка по комментарию администратора, контакты куратора |
| **Админ-панель** | Пароль из `.admin_password` или `ADMIN_PASSWORD`; фильтры и поиск; статусы: на проверке, доработка, одобрено, отклонено, резерв; редактирование полей; выгрузка Excel (SheetJS, отдельный чанк) |
| **Служебная страница** | `/yandexouth` — OAuth Яндекса для получения токена загрузки на Диск (скрытый маршрут SPA) |

## Стек технологий

| Категория | Технологии |
|-----------|------------|
| **Frontend** | React 18, TypeScript, Vite 6, Tailwind CSS 4 (глобальные стили), inline-стили в компонентах |
| **Сборка** | `tsconfig.json`: алиас `@/*` → `./src/*` |
| **Backend** | PHP в `public/api/` → при `npm run build` копируется в `dist/api/` |
| **База данных** | MySQL (пользователи, заявки); подключение через `api/db_config.php` и `api/.db_credentials` |
| **Файлы** | Яндекс.Диск (`upload_to_yandex.php`, структура `top/…/смена/ФИО`) или локально `api/uploads/` |
| **Excel** | Библиотека `xlsx` (динамический импорт в админке, отдельный JS-чанк) |

## Хранение данных

1. **MySQL** — основной источник правды: таблицы `users`, `applications` (в т.ч. статус `reserve`, льготы JSON, вложения).
2. **localStorage** — `top_user`, `top_auth_token` для сессии пользователя после регистрации/входа; дублирование части полей для быстрого UI.
3. **Legacy** — `api/save_data.php` и `api/data/applications.json` (если используются в старом сценарии); актуальный поток — через `register.php` и БД.
4. **Файлы заявок** — пути в JSON-поле `attachments` у заявки; физически на Диске или в `api/uploads/`.

## API (PHP), основные точки

| Файл | Назначение |
|------|------------|
| `register.php` | Регистрация, создание пользователя и заявки в БД |
| `login.php` | Вход, выдача Bearer-токена |
| `get_applications.php` | `mode=my` — заявка пользователя; `mode=all` + пароль админа — все заявки |
| `update_application.php` | Обновление статуса/полей (пользователь или администратор) |
| `get_stats.php` | Публичная статистика по сменам для блока «Свободные места» |
| `upload.php` / `upload_to_yandex.php` | Загрузка файлов (по одному файлу за запрос на клиенте) |
| `get_yandex_file.php` | Ссылки на скачивание с Диска |
| `yandex_oauth_config.php`, `yandex_oauth_callback.php` | OAuth Яндекса |
| `db_setup.php` | Однократное создание/миграция таблиц (**удалить с продакшена после выполнения**) |
| `save_data.php` | Legacy-сохранение в JSON (при необходимости) |

Заголовок `Authorization` на Apache пробрасывается через корневой `public/.htaccess` (копируется в `dist`).

## Льготные категории (ID в форме)

Используются как строковые ID в JSON поля `benefits`:

| ID | Описание в интерфейсе |
|----|------------------------|
| `low_income` | Малоимущие |
| `svo` | Дети участников СВО |
| `orphan` | Дети-сироты |
| `disabled` | Дети-инвалиды |
| `ovz` | Дети ОВЗ |
| `combat_veteran` | Ветераны БД |
| `kmns` | КМНС |
| `preventive` | Профилактический учёт |
| `large_family` | Многодетные семьи |
| `none` | Без льгот |

Выбор «Без льгот» взаимоисключающий с другими категориями.

## Документы для загрузки (форма)

Порядок и подписи в UI; **пункты 1–11 обязательны** для отправки заявки (12-й — «Иные документы», необязателен):

1. Паспорт ребёнка (при наличии)  
2. Свидетельство о рождении ребёнка  
3. СНИЛС  
4. ИНН  
5. Справка с места учёбы  
6. Подтверждение льготной категории  
7. Справка об отсутствии судимости (или подтверждение заказа справки)  
8. Приписное удостоверение для юношей  
9. Заявление в межведомственную комиссию  
10. Согласие на обработку персональных данных  
11. Банковские реквизиты  
12. Иные документы (необязательно)  

Шаблоны для заполнения: [папка на Яндекс.Диске](https://disk.yandex.ru/d/tq_-HiC6wB7wRg).

## Локальная разработка

```bash
npm install
npm run dev
```

Откройте адрес из консоли Vite (обычно `http://localhost:5173`). Запросы к `/api/*` в dev идут на тот же origin; для работы PHP удобнее собрать проект и поднять `php -S` на `dist` (см. ниже).

### Сборка

```bash
npm run build
```

В **`dist`**: `index.html`, `assets/` (в т.ч. основной бандл и отдельный чанк для `xlsx`), **`api/`** — копия `public/api/`, **`.htaccess`** из `public/` — в корне `dist` для SPA и Authorization.

### Просмотр сборки с PHP

```bash
php -S localhost:8080 -t dist
```

Откройте `http://localhost:8080`. Настройте БД и `.db_credentials` в `dist/api/` при необходимости.

## Безопасность (кратко)

- Пароли в БД — через хеширование (см. `register.php` / `login.php`).
- Секреты не коммитить: `.db_credentials`, `.yandex_oauth_token`, `.yandex_app_credentials`, `.admin_password` — в `.gitignore`.
- `api/uploads/` — `.htaccess` против выполнения скриптов.
- Продакшен: HTTPS, после миграции удалить `db_setup.php` с сервера.
